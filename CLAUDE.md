# 체크앤고 (Check & Go) — 프로젝트 메모리

청년 정책 매칭 서비스. 내 조건에 맞는 정부 지원 정책을 필터링해서 보여주고, 서류 체크리스트와 마감 캘린더로 신청까지 챙겨주는 웹 서비스.

기획 문서(정책 정의서, IA, 화면플로우, 데이터구조, API명세, PRD)는 `/docs` 참고. 여기엔 코드 작성 시 매번 다시 설명하지 않아도 되는 것만 남김.

## 관련 문서

자세한 기획 배경(트렌드/니즈, 페르소나, 정책 정의서, IA, 화면플로우, 데이터구조, MVP 정책 큐레이션, API명세, PRD)은 `docs/프로젝트_로드맵.md` 참고. 이 파일은 자동으로 로드되지 않으니, 기획 의도나 세부 규칙이 궁금할 때 직접 "docs/포트폴리오_프로젝트_로드맵.md 읽어줘"라고 요청할 것.

## 기술 스택

- Next.js + TypeScript + Tailwind
- Supabase (DB + Auth, 이메일/비밀번호 로그인만 사용, 소셜로그인 없음)
- Resend (이메일 발송, 마감 7일전·1일전 리마인더)
- Vercel Cron Jobs (1일 1회 배치로 온통청년 API 동기화 — 무료 플랜 제약과 일치하니 더 잦은 주기로 바꾸려 하지 말 것)
- react-day-picker (캘린더 UI)
- 배포: Vercel

## 핵심 데이터 규칙 (자세한 건 정책 정의서 참고)

- 정책 자격조건(나이/취업상태/주거형태)은 배열로 저장. `"제한없음"`은 항상 단독 값으로만 넣고, 다른 값과 섞지 말 것
- 소득조건은 `has_income_condition`(boolean)만 구조화. 정확한 기준 문구는 `description`에 원문 그대로
- 나이는 `birth_date`(생년월일)로 입력받아 서버에서 만 나이 계산 후 구간 매핑. 프론트에서 구간을 직접 선택하게 하지 말 것
- `deadline_type`이 "상시"인 정책은 `deadline`이 null. 이 경우 D-day·마감임박 정렬·캘린더 표시 로직에서 제외
- 마감 여부는 자격조건 매칭(AND 로직)과 완전히 별개. 마감돼도 매칭 자체는 그대로 성립하고, 화면에서 반투명 처리만 함
- 마이페이지에는 스크랩한 정책만 노출. 매칭된 전체 정책이 아님

## 기준일 고정 (freeze) — 중요

데이터 수집 마감(2026-08-15) 이후로는 정책 데이터를 갱신하지 않음. 마감 여부·마감임박 정렬·캘린더에 쓰는 "오늘 날짜"는 실제 시스템 날짜가 아니라 **고정된 기준일 상수**를 써야 함 (예: `REFERENCE_DATE`). `new Date()`를 이 목적으로 직접 쓰지 말 것 — 배포 후 시간이 지나면 모든 정책이 마감으로 표시되는 버그가 생김. 단, `created_at` 같은 실제 타임스탬프 기록은 정상적으로 시스템 날짜 사용.

## 보안

- 온통청년 API 인증키는 서버(백엔드)에서만 사용. 프론트엔드 코드, 환경변수 중 `NEXT_PUBLIC_` 접두사가 붙는 곳에 절대 넣지 말 것
- Supabase RLS(Row Level Security) 활성화 상태 유지. 우회하는 코드 작성 금지

## 코드 컨벤션

- 컴포넌트: PascalCase, 파일명도 동일
- 폴더 구조: `app/`, `components/`, `lib/`(matching 로직 등 비즈니스 규칙), `types/`
- 커밋 전 `npm run lint`, `npm run build` 통과 확인
